# Кубок Авесто-2026 — Инструкция по запуску

## Быстрый старт

```bash
cd "/Users/abdusalom.rabikhov/Claude projects/tournament"
npm install
npm run dev
```

Откройте браузер: **http://localhost:3002**

Бэкенда нет — все данные (команды, матчи, игроки, награды) хранятся прямо в
localStorage браузера. Никакой базы данных, никакого сервера для данных.

---

## Первые шаги

1. Откройте сайт — при первом заходе автоматически создаётся стартовый
   набор данных (18 команд, логотипы, расписание группового этапа) из
   `src/data/seed.ts`.
2. Зайдите в админку: кнопка **«Админ»** в шапке сайта → `/admin`.
3. Пароль администратора — см. `src/config.ts`.
4. Управляйте турниром: команды и логотипы, составы игроков, матчи и
   результаты, награды, дата обратного отсчёта на главной.

---

## Важно: где хранятся данные

**Единственный источник правды на старте — `src/data/seed.ts`.** Все правки,
сделанные через админку в браузере (переименование команд, загрузка
логотипов, создание матчей), живут **только в localStorage конкретного
браузера**. Они:

- не сохраняются в git,
- не видны в режиме инкогнито,
- не видны на другом компьютере/браузере,
- **теряются**, если открыть сайт в новой вкладке инкогнито или в другом
  профиле — покажется старый seed, а не то, что вы настроили.

Если правки в админке должны стать «постоянными» (видны всем, переживают
инкогнито, переживают сброс localStorage) — их нужно **перенести в код**,
в `src/data/seed.ts`. Порядок: экспортировать текущее состояние из
localStorage → передать значения ассистенту → он обновит `seed.ts` →
задеплоить. См. `MEMORY.md`, раздел «Единственный источник правды по
командам», там же готовая консольная команда для экспорта.

---

## Сброс данных

Если данные испортились или нужно вернуться к чистому стартовому состоянию:

- В админке: `/admin/settings` → «Пересобрать расписание» (пересобирает
  только матчи группового этапа, не трогает команды/логотипы/составы).
- Полный сброс: очистить localStorage браузера для `localhost:3002`
  (DevTools → Application → Local Storage → Delete) и обновить страницу —
  подтянется свежий `seed.ts`.

## Стадии турнира

Групповой этап (4 группы, круговая система) → плей-офф (1/4 финала →
полуфинал → финал + матч за 3-е место). Топ-2 каждой группы выходят в
плей-офф. Сетка плей-офф на `/bracket` строится автоматически из текущей
турнирной таблицы — обновляется сама по себе по мере ввода результатов.
