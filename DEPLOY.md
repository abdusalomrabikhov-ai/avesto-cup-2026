# Деплой «Кубок Авесто-2026»

Стек: React 19 + Vite + TypeScript + Tailwind CSS v4. Без бэкенда, без базы
данных — статическая сборка (`npm run build` → `dist/`), данные турнира
живут в localStorage браузера каждого посетителя (см. `ЗАПУСК.md` про то,
что это значит для команд/логотипов).

## Сборка

```bash
npm run build
```

Собирает в `dist/` (`tsc -b && vite build`). Проверить локально:

```bash
npm run preview
```

## Куда деплоить

Любой статический хостинг подойдёт — сайт не требует сервера, только
раздачу файлов:

- **Vercel** — `vercel deploy`, или подключить репозиторий, build command
  `npm run build`, output directory `dist`.
- **Netlify** — build command `npm run build`, publish directory `dist`.
- **Cloudflare Pages** — то же самое, build command/output идентичны.

## Перед деплоем

- `npx tsc --noEmit -p tsconfig.app.json` — typecheck чистый
- `npx vitest run` — тесты зелёные (standings/playoff/forfeit/scorers)
- `npm run build` — сборка без ошибок
- Если менялись команды/логотипы через админку локально — **они не попадут
  в деплой сами по себе**. Их нужно сначала перенести в `src/data/seed.ts`
  (см. `CLAUDE.md`/`MEMORY.md`), иначе задеплоенный сайт покажет старые
  названия/логотипы всем новым посетителям.

## После деплоя

Открыть сайт **в режиме инкогнито** (или на другом устройстве) и сверить
названия команд/логотипов с тем, что должно быть — это единственный
надёжный способ увидеть именно то, что видят реальные посетители (не то,
что накопилось в localStorage вашего собственного браузера за время
разработки).

## Известные ограничения сборки

- Основной JS-бандл >500KB (предупреждение esbuild/rollup) — за счёт видео
  в hero-секции лендинга (`src/assets/brand/hero-video.mp4`, ~3.7MB) и React
  runtime. Не критично для статического сайта, но при желании оптимизации:
  code-splitting через `dynamic import()` или вынос видео в отдельный CDN.
