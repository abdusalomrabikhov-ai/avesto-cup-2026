# Промт для Claude Code — сайт турнира «Кубок Авесто-2026»

Создай сайт для баскетбольного турнира **«Кубок Авесто-2026»** среди корпоративных команд ООО «Авесто Групп».

**Стек:** React + Vite + TypeScript, Tailwind CSS. Без бэкенда — данные в localStorage + стартовый JSON. Логотипы и фото игроков храни как base64 в localStorage. Запуск: `npm install && npm run dev`.

## Структура турнира (строго по регламенту)

1. **Групповой этап.** 4 группы — A, B, C, D. Внутри группы играют по круговой системе «каждый с каждым».
2. **Плей-офф.** Выходят команды, занявшие 1 и 2 места в каждой группе (8 команд). Этапы: полуфиналы → матч за 3-е место → финал. При ничьей в плей-офф — овертайм 5 минут (можно вносить несколько овертаймов до победителя).

## Начисление очков в группе

Победа = 2 очка, поражение = 1 очко, неявка = 0 очков. Техническое поражение при неявке — счёт 0:20 (в админке галочка «неявка», счёт проставляется автоматически).

## Тай-брейк при равенстве очков (каскад сравнений именно в этом порядке)

1. результат личной встречи;
2. разница заброшенных/пропущенных очков в личных встречах;
3. общая разница заброшенных и пропущенных очков;
4. общее количество заброшенных очков;
5. жребий (ручной флаг в админке).

## Составы команд и игроки (по регламенту)

- У каждой команды есть заявочный состав: **не менее 5 и не более 12 игроков** (валидация в админке).
- Поля игрока: ФИО, дата рождения, должность в компании, игровой номер, **фото (base64)**, отметка «капитан» (у капитана — контактный телефон).
- Цвет игровой формы команды (поле у команды).
- Один игрок не может быть заявлен за две команды (валидация по всему турниру).

## Статистика игроков

При вводе результата матча можно (опционально) внести по игрокам набранные очки. На основе этого автоматически считается бомбардирский рейтинг. Показатели «защитник»/«лучший игрок» задаются вручную (в регламенте — по решению оргкомитета).

## Страницы

1. **Главная — Групповые таблицы.** 4 таблицы (A/B/C/D). Столбцы: Место, Логотип+Команда, И, В, П, ЗО, ПО, Разница ±, Очки. Первые 2 места в каждой группе подсвечены как «выход в плей-офф».
2. **Сетка плей-офф.** Наглядный bracket: полуфиналы, матч за 3-е место, финал. Счёт, победитель, овертаймы отдельно.
3. **Матчи.** Все игры с фильтром по группе/стадии: дата, команды, счёт, статус (запланирован / завершён / техническое поражение).
4. **Страница команды.** Логотип, группа, цвет формы, состав игроков (карточки с фото, номером, должностью, капитан отмечен), статистика, история матчей, форма (последние игры W/L).
5. **Страница игрока / рейтинг бомбардиров.** Таблица лидеров по очкам, переход на профиль игрока.
6. **Индивидуальные номинации.** Блок наград: «Лучший игрок турнира», «Лучший защитник турнира», «Лучший бомбардир турнира» — назначаются вручную в админке (бомбардир может подсказываться из рейтинга).

7. **Админ-панель (`/admin`, вход по паролю из конфига):**
   - управление командами: название, компания/подразделение, цвет формы, **загрузка логотипа** (input file, превью, base64), распределение по группам A/B/C/D;
   - управление составом: добавление/редактирование игроков (ФИО, дата рождения, должность, номер, фото, капитан+телефон), валидация 5–12 игроков и уникальности игрока в турнире;
   - создание матчей и ввод результатов (общий счёт + опционально очки по игрокам) — таблицы, сетка и рейтинг бомбардиров пересчитываются автоматически;
   - отметка «неявка» → авто-счёт 0:20;
   - ввод овертаймов для плей-офф;
   - ручной флаг «жребий» для разрешения полного равенства;
   - назначение победителей индивидуальных номинаций.

## Логика

Вынеси в чистые функции построение таблицы группы (полный каскад тай-брейков), разрешение сетки плей-офф, авто-счёт неявки и агрегацию очков игроков в рейтинг бомбардиров. Покрой юнит-тестами (Vitest): тай-брейки, неявка 0:20, суммирование очков игроков.

## Модель данных

- **Team** (id, name, company, group, jerseyColor, logo)
- **Player** (id, teamId, fullName, birthDate, position, number, photo, isCaptain, phone)
- **Match** (id, stage, group, teamA, teamB, scoreA, scoreB, status, isForfeit, overtimes[], playerStats[])
- **PlayoffStage**
- **Award**

## Дизайн

Современный спортивный, тёмная тема с яркими акцентами, адаптив (моб + десктоп), крупные читаемые таблицы, иконки lucide-react. Брендинг «Кубок Авесто-2026» в шапке.

## Демо-данные

16 команд (по 4 в группах A/B/C/D), у каждой 5–8 игроков с номерами и должностями, частично сыгранные матчи с очками игроков — чтобы таблицы, часть сетки и рейтинг бомбардиров сразу были заполнены.
